# 設計メモ（UI/UX）

## ナビゲーション / 情報構造
- Vue Router を使い、`/` をトップ、`/tools/:id` で各ツールを分割（下記ルーティング案参照）。
- ヘッダー: ロゴ（Home リンク）＋ドロップダウン/スライドインでツール一覧。モバイルで片手操作できる位置に配置。
- 各ツールページは縦 1 カラム構成、ファーストビューに入力欄と主要結果を配置。

## レイアウト
- モバイル幅 360–430px を基準にデザイン。PC でも 480–560px に最大幅を固定。
- ページ余白は左右 12–16px、カード間 12px 程度。
- ボタンと入力はタップしやすい高さ 44px 以上。
- セクション: ページ見出し h1、主要ブロック h2/h3。縦間隔は 12–16px。
- カード/コンテナ: パディング 12–16px、ページ上部余白 16–20px。

## ビジュアル
- 現行のグラデ/カードスタイルを維持しつつ、モバイルで読みやすいタイポサイズ（14–16px）。
- 背景は淡いグラデ、カードは角丸＋軽いシャドウ。
- 強調色はブルー系、警告/情報はタグ/バッジで簡潔に表示。

## 各ページの構成（概要）
- Home: ツール一覧カード、概要、使い方リンク。
- 点数計算/符計算: 翻/符入力と支払い結果。符計算の補足リンク。
- 待ち牌/受け入れ: 手牌入力＋待ち表示。14 枚時の簡易何切る提案。
- 役一覧: 検索ボックス＋タグフィルタ＋役カード。
- 点差逆転: 点数入力＋加点必要例。
- 何切る練習: 問題選択＋選択ボタン＋フィードバック。JSON からの問題読込は将来拡張。
- 牌理可視化: 手牌入力→刻子/順子/対子/孤立牌を色分け表示。
- 早見表: 翻スライダーで符ごとの点数表。
- リーチ vs ダマ: 翻/符＋和了率/放銃率入力で簡易 EV 差分。
- 点棒管理: 4 人の点数増減ボタン。スマホで押しやすいサイズ。
- 牌譜ビューア: JSON ペースト→1 ステップずつ再生。

## テキストワイヤー（モバイル 1 カラム）
- Home

  - オフライン注意/使い方のミニセクション

  - 結果カード: 親ロン/子ロン/親ツモ/子ツモの点表示＋満貫ラベル

  - 入力: 手牌テキスト + 牌ボタン（グリッド）
  - 牌数表示と手牌プレビュー
  - 待ちリスト（牌＋残り枚数）、合計受け入れ
  - 14枚時の簡易何切る提案
- 役一覧
  - 検索入力、タグフィルタ（ピルボタン）
  - 役カード（役名/翻/条件/タグ）
- 点差逆転
  - 点数入力（自分/相手）、親子/ツモロン切替
  - 必要加点表示、翻/符候補リスト
- 何切る練習
  - 問題セレクタ（select）
  - 手牌表示/シナリオ
  - 選択肢ボタン（4択想定）
  - フィードバック（正解/理由）＋機械判定の受け入れ候補
- 牌理可視化
  - 手牌入力
  - 刻子/順子/対子/孤立牌のグループ表示
- 早見表
  - 翻スライダー
  - 符ごとの点数テーブル（親ロン/子ロン/ツモ配分）
- リーチ vs ダマ
  - 入力: 翻/符、親子切替、和了率スライダー、放銃率スライダー、失点入力
  - 出力: ダマEV、リーチEV、差分タグ
- 点棒管理

  - 増減ボタン（+/- 任意額、+/-1000）
  - リセットボタン

  - JSON入力（textarea）
  - タイトル/ステップ表示

  - 全アクションのリスト

## ルーティング案（モバイル前提）
- `/` : Home（ツール一覧）
- `/tools/score`
- `/tools/ukeire`
- `/tools/yaku`
- `/tools/endgame`
- `/tools/practice`
- `/tools/shape`
- `/tools/ref`
- `/tools/ev`
- `/tools/tracker`
- `/tools/replay`
各ルートは遅延 import で読み込み、meta に `title`（ページタイトル）を付与。

## 共通レイアウト
- ヘッダー: Home リンク + ハンバーガー/ドロップダウンでツール一覧。
- フッター: 簡易リンク（Home / ライセンス / オフライン注意など）。
- コンテナ幅: max 560px、左右 12–16px の余白。
- スクロール: ルート遷移時にトップへスクロール。
- タイトル: ルート meta.title を `<title>` に反映。
- メニュー: ヘッダーにツール一覧を開くメニュー（ドロワー/ドロップダウン）。Home へ戻る導線を常設。

## 入力コンポーネント仕様（方針）
- 牌入力: テキスト + 牌ボタン（タップで追加）。14/13枚制限を表示。ドラッグ操作なし。
- 数値入力: `type=number` か `range` を併用。最小/最大/ステップを明記し、バリデーションする。
- スライダー: 翻/和了率/放銃率などの概算入力に使用。ラベルで現在値を隣接表示。
- フォーカス: フォーカスリングを隠さず、タップ/キーボード双方で操作可能にする。
- aria/label: 入力にはラベルを付け、必要に応じて `aria-label` を補足。

## テーマ切替（方針）
- 基本色をトークン化（例: `--accent`, `--bg`, `--card`, `--text`）。テーマ定義を配列で持ち切替。
- 軽量なテーマセレクタをヘッダー/メニューに配置。選択は localStorage に保存（オフライン対応）。
- 初期テーマは現行ブルー系。追加候補: Mリーグチーム風のカラーパレット。
- トークン例: `--accent`, `--accent-2`, `--accent-contrast`, `--bg`, `--bg-soft`, `--card`, `--text`, `--muted`。
- パレット候補: ブルー基調、レッド基調、グリーン基調（Mリーグ風）。

## 入力/操作ガイド
- 牌入力はテキストとタップ併用。ドラッグ操作は不要。
- 長い説明は折りたたみでまとめ、主要結果を先頭に表示。
- フォーカスリングは保持し、aria-label/label を必須にする。

## 遷移/メニュー動線（テキスト）
- Home から各 `/tools/:id` へカード/メニューで遷移。
- ヘッダーのメニューから全ツールページへ直接遷移可能。
- フッターの Home リンクでトップへ戻る。
- ルート遷移時はスクロールトップ + `<title>` 更新。
